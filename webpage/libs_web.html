Instructions to Create and Configure SSH Key (Github/Gitlab)

  1. Open Terminal:
    • Open the GitBash application on your computer.
  2. Generate a New SSH Key:
    • To generate a new SSH key, enter the following command: 
      ssh-keygen -t rsa -b 4096 -C "your_email@example.com"
    • When prompted, press Enter to save the key to the default location (/Users/your_username/.ssh/id_rsa).
    • You can set a passphrase for added security or press Enter to skip this step.
  3. Copy SSH Key to Clipboard: 
    • cat ~/.ssh/id_rsa.pub
  4. Add SSH Key to GitHub Account:
    • Navigate to Git Site, click on your profile, and select Preferences.
    • In the left sidebar, click SSH keys, then click Add new key.
    • Paste your SSH key into the Key field, give it a Title, and click Add key.

Renaming a Single Word of All the Files in the Folder (UNIX/Linux)

find /folderpath/foldername -type f -name '*OLD_NAME*' -exec rename OLD_NAME NEW_NAME {} +

Description of the Command:

This command is used to rename a specific word or pattern in the names of all files within a specified directory in a UNIX/Linux environment. It utilizes the find command to locate the files and the rename command to modify their names.

  1. find /folderpath/foldername: Initiates the find command, which searches for files and directories within the specified directory. Replace /folderpath/foldername with the actual path where you want to search.
  2. -type f: Restricts the search to files only, excluding directories.
  3. -name '*OLD_NAME*': Specifies the pattern of the file names to search for, using wildcard characters (*) to match any files containing OLD_NAME in their names. Replace OLD_NAME with the specific word or pattern you want to rename.
  4. -exec rename OLD_NAME NEW_NAME {} +: Executes the rename command on each file found by find:
    1. rename: The command used to rename files.
    2. OLD_NAME: The existing word or pattern in the file names to be replaced.
    3. NEW_NAME: The new word or pattern to replace OLD_NAME in the file names.
    4. {}: A placeholder representing each file found by the find command.
    5. +: Executes the rename command for all matching files at once (in batches), improving efficiency.

By using this command, you can efficiently rename a specific word in multiple files at once, saving time and effort compared to manually renaming each file.

Develop an Executable File (.exe) using UNIX/Linux Shell

pyinstaller --onefile --noconsole --distpath "C:/PATH/build/" --workpath "C:/PATH/build/" --specpath "C:/PATH/build/" main.py

Description of the Command:

This section provides a guide on how to convert a Python script into a standalone executable file using the PyInstaller tool in a UNIX/Linux environment. This can be useful for distributing Python applications to users who do not have Python installed on their systems.

  1. pyinstaller: This is the command used to initiate the conversion of the Python script into an executable file.
  2. --onefile: This option packages everything into a single executable file, which simplifies distribution.
  3. --noconsole: This option suppresses the console window that would normally open when running the executable, making it ideal for GUI applications.
  4. --distpath "C:/PATH/build/": Specifies the output directory for the final executable file. Replace "C:/PATH/build/" with your desired path.
  5. --workpath "C:/PATH/build/": Specifies the working directory where PyInstaller stores its temporary files. Again, replace "C:/PATH/build/" with your desired path.
  6. --specpath "C:/PATH/build/": Sets the path for the .spec file, which PyInstaller uses to remember the configuration for the build. Replace "C:/PATH/build/" accordingly.
  7. main.py: This is the name of the Python script you want to convert. Replace main.py with your actual script file.

After setting the correct paths and ensuring PyInstaller is installed, run this command in your shell to generate the executable file.

Instructions to Change System Language to Japanese

  1. Search for Language Settings: 
    • In the Windows Search Box (⌘ + S), search for Language.
  2. Open Language Settings: 
    • From the search results, select Language settings under the Settings menu.
  3. Install Japanese Language: 
    • In the Preferred Languages section, click on Add a language.
                    
    • Search for Japanese and follow the prompts to install it.
  4. Access Administrative Language Settings: 
    • In the same Language settings menu, navigate to Related settings.
                    
    • Click on Administrative language settings.
  5. Change System Locale: 
    • In Administrative, under the Language for non-Unicode programs section, click on Change system locale.
                    
    • From the dropdown menu, select Japanese (Japan).
  6. Restart Your PC: 
    • Confirm the changes and restart your computer for the changes to take effect.
